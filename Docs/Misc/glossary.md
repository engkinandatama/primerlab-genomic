# **PrimerLab — Glossary of Terms**

This glossary defines all domain-specific and project-specific terminology used throughout PrimerLab’s documentation, codebase, and workflows.

Terms are categorized into:

* Bioinformatics Terms
* Workflow & QC Terms
* Software Architecture Terms
* Output & Reporting Terms
* Config & CLI Terms
* Testing Terms

This ensures all contributors share a unified vocabulary.

---

# **1. Bioinformatics Terms**

### **Amplicon**

A DNA fragment generated by PCR amplification, defined by forward + reverse primer boundaries.

### **Primer**

A short oligonucleotide that binds to a DNA/RNA target sequence to initiate polymerase extension.

### **Forward Primer / Reverse Primer**

Primer pair defining the 5' and 3' boundaries of an amplicon.

### **GC Content**

Percentage of bases G+C in a sequence. Affects Tm and stability.

### **Melting Temperature (Tm)**

Temperature where 50% of DNA duplex dissociates. Critical for primer selection.

### **Self-Dimer / Cross-Dimer**

Unwanted interaction between primers forming dimers instead of binding target sequence.

### **Hairpin**

Secondary structure where a primer folds back on itself.

### **Primer3**

External tool used for primer design; optional in PrimerLab.

### **BLAST**

External tool for sequence similarity search; used for off-target analysis.

### **Off-Target**

Unintended hybridization site for a primer or CRISPR guide.

### **PAM (Protospacer Adjacent Motif)**

Short sequence essential for CRISPR target recognition.

### **qPCR Probe**

Fluorescent oligonucleotide used in real-time PCR assays.

### **Multiplex PCR**

PCR using multiple primer sets simultaneously.

### **ViennaRNA**

Toolset for predicting RNA secondary structure.

---

# **2. Workflow & QC Terms**

### **Workflow**

A structured sequence of analytical steps executed end-to-end.

### **QC (Quality Control)**

Validation stage checking primer, probe, or guide quality metrics.

### **QC Engine**

Internal module calculating QC metrics (hairpin, dimer, Tm ranges, etc.).

### **Candidate Set**

Collection of all possible primers produced before QC filtering.

### **Heuristics**

Rule-based scoring used in primer selection.

### **Mismatch Tolerance**

Allowable mismatches in CRISPR off-target scoring.

### **ΔG (Delta G)**

Free energy metric used in structure stability evaluation.

### **Thermodynamic Model**

Model predicting primer structural behavior and stability.

---

# **3. Software Architecture Terms**

### **Public API**

Functions users can import directly from `primerlab.api`.

### **Workflow Layer**

Orchestration logic coordinating core calculations.

### **Core Layer**

Fundamental utilities:

* sequence functions
* config loader
* error system
* tools wrappers
* data models
* logging system

### **Data Model**

A structured object representing biological or workflow results, implemented as dataclasses.

### **WorkflowResult**

Standardized output container for all workflows.

### **Error Code**

Stable machine-readable identifier for errors (e.g., `ERR_CONFIG_001`).

### **Exception Layer**

Custom exceptions internal to PrimerLab.

### **Config Loader**

System merging YAML + CLI overrides into a validated dictionary.

### **Output System**

Component responsible for generating:

* folders
* JSON
* Markdown reports
* debug files

### **Logger**

Centralized multi-level logging system.

### **Progress System**

Reports step-by-step execution progress to CLI & logs.

### **Tool Wrapper**

Secure, deterministic wrapper around external binaries (Primer3, BLAST).

---

# **4. Output & Reporting Terms**

### **Run Folder**

Dedicated directory for a single workflow execution.

### **JSON Results**

Machine-readable export of WorkflowResult.

### **Markdown Report**

Human-readable analysis summary for the run.

### **Debug Folder**

Contains raw tool outputs and traceback for debugging.

### **Run Metadata**

JSON file describing:

* environment
* tool versions
* PrimerLab version
* timestamp
* workflow parameters

### **Golden Output**

Reference output used for regression testing.

---

# **5. Config & CLI Terms**

### **YAML Configuration File**

Primary source of workflow settings.

### **CLI Override**

Command-line options that adjust or override YAML.

### **Merged Config**

Final configuration after merging YAML + CLI parameters and validation.

### **Workflow Parameter Schema**

Definition of valid keys and types for a workflow config.

### **Config Validation**

Checks ensuring:

* fields exist
* types correct
* ranges valid
* optional tools enabled safely

### **CLI Command / Subcommand**

User-facing entrypoint, e.g.:

```
primerlab pcr --config path.yaml
```

---

# **6. Testing Terms**

### **Unit Test**

Tests a single function or module in isolation.

### **Integration Test**

Validates end-to-end behavior.

### **Regression Test**

Ensures new code does not change existing behavior unexpectedly.

### **Golden Test**

Compares output to previously approved “golden” files.

### **Mock**

Artificial substitute for external tools or heavy functions.

### **Test Fixture**

Reusable test data or setup functions.

### **Test Isolation**

Ensuring tests do not depend on external state or network.

### **Determinism**

Guarantee that the same input always yields the same output.

---

# **7. Project Governance Terms (Internal)**

### **ADR (Architecture Decision Record)**

Formal record of a major design decision.

### **Contributor**

Anyone modifying PrimerLab, including automated contributors.

### **Stability Contract**

Promise to avoid breaking:

* public API
* config schema
* output structure
* error codes

### **Backward Compatibility**

Ability to use older configurations and scripts with newer versions.

---

# **8. Summary**

This glossary defines all essential terminology used across PrimerLab:

* biological terms (amplicon, Tm, hairpin, PAM)
* workflow and QC terms
* core architecture vocabulary
* output/report artifacts
* config & CLI concepts
* testing frameworks
* project governance terms

This ensures all contributors—human or automated—share a consistent understanding of PrimerLab’s conceptual model.
